# 内部三档基准（Anthropic 官方，同 harness 只变档位）

**这是唯一一次测出档位对正确率的影响**，而且它落在我们已经立了铁律的地方。

## 方法

- 臂：`claude -p --model haiku|sonnet|opus`（Haiku 4.5 / Sonnet 5 / Opus 4.8），`--output-format json` 取结构化 usage
- 同一厂商、同一 harness，**唯一变量是档位** —— 排除了跨 CLI 的人格、错误契约、传参差异
- 任务：4 个自造硬任务（同 `RESULTS-HARD-20260708.md`）；每 trial 唯一 nonce；prompt 走 stdin
- n=3，取中位数

## 真实成本（用核实过的单价）

单价（Claude API，2026-06-24）：Haiku 4.5 `$1/$5`；Sonnet 5 `$3/$15`（引导价 `$2/$10` 至 2026-08-31）；Opus 4.8 `$5/$25`（$/M tokens）。缓存读 ≈ 0.1× 输入价。

估算式：`fresh × 输入价 + cache_read × 0.1 × 输入价 + output × 输出价`

| 任务 | haiku | sonnet | opus |
|---|---|---|---|
| h-interval | $0.0472 (1.00×) | $0.0586 (1.24×) | $0.0861 (1.82×) |
| h-checksum | $0.0076 (1.00×) | $0.0505 (6.63×) | $0.0754 (**9.90×**) |
| h-recur | $0.0550 (1.00×) | $0.2623 (4.77×) | $0.1621 (2.95×) |
| h-ambig | $0.0339 (1.00×) | $0.0545 (1.61×) | $0.0807 (2.38×) |
| **合计** | **$0.1436 (1.00×)** | $0.4258 (2.97×) | $0.4042 (2.82×) |

### 三条结论

1. **便宜档确实便宜约 3 倍 —— 输出多 12 倍也没吃掉价差。**
   haiku 单价低 5 倍，且其 `fresh` 近乎全被缓存吸收（fresh=10）。**"默认往最低档派"在经济上成立。**

2. **成本在档位上非单调。** 标准价下 sonnet（2.97×）比 opus（2.82×）还贵，因为它在 h-recur 上 thrash 到 47.6 万 input。
   **⚠️ 但这条依赖定价：** 按 Sonnet 5 的引导价 `$2/$10`，sonnet 降到约 1.98×，反而比 opus 便宜。
   **稳健的说法是 token 行为，不是美元排序**：中档 thrash 最狠（9 轮 vs opus 2 轮），美元排序会随定价翻转，2026-08-31 引导价结束后再复核。

3. **haiku 唯一的正确率失败，落在已被铁律禁止的路由上。**
   需要模型亲自算 → 让它写代码 / 编排者核验。遵守铁律时，haiku 在全部四个硬任务上与 opus 持平，且便宜 3 倍。

### 升级阶梯的一条修正

若必须从最低档升级，**别默认升到中档**。本数据里中档在 token 消耗上最差（thrash 最重）。先按 SKILL.md 阶梯做零代价的两步（查输入是否送达 / 改成让模型写代码），仍不行再直接考虑高档。

## 本数据【不能】支持的

- n=3、4 个任务、单一厂商，方差大。
- 成本估算未分离 `cache_creation`（计 1.25×）与 `cache_read`（0.1×），按 0.1× 近似 → **略低估 sonnet/opus**（它们 fresh 更高），真实差距只会更大。
- 未测 `effort` 旋钮（`claude -p` 似乎不暴露 `--effort`）。
- 自造任务只缓解、不能证明消除训练集污染。
- 跨厂商成本仍不可比（订阅制无美元口径；此处能算美元是因为 Anthropic 官方单价公开）。
