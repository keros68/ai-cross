**原本自称是本项目方法学最干净的一次测试**——5 个模型全走同一个裸 API 端点（OpenAI 兼容），
无 harness、无 CLI 人格污染、prompt 走 JSON body 不会被截断、我完全控制输入。
跨模型终于是真·同条件对比。但样本仍小（n=3，7 任务），结论需按此读。

## 方法

- 端点：`api.siliconflow.cn/v1/chat/completions`（纯按量，非 coding plan）
- 模型：Qwen3.6-35B-A3B / GLM-5.2 / DeepSeek-V4-Pro / Kimi-K2.6 / LongCat-2.0
- 任务（全自造、程序化 ground truth）：抽取金额、情感分类×3、日期归一化、数句号、JSON 抽取
- verifier：15 项正/负样本自检通过（含"带思考残留""带围栏""顺序无关"等真实场景）
- max_tokens=512，**默认思考开**（out-of-box）；content 空回退 reasoning 尾 200 字
- 记正确率 + prompt/completion token + 耗时

## 结果

| 模型 | 正确率 | prompt 中位 | completion 中位 | 秒中位 |
|---|---|---|---|---|
| Qwen3.6-35B | 86%* | 48 | **426** | 8.2 |
| GLM-5.2 | 100% | 50 | 215 | 4.2 |
| DeepSeek-V4-Pro | 100% | 42 | **40** | 3.0 |
| Kimi-K2.6 | 100% | 43 | 117 | 3.2 |
| LongCat-2.0 | 95% | 44 | 127 | 4.9 |

各任务正确率：除 `extract-amounts`（Qwen 0%*、LongCat 67%，余 100%）外，
**其余 6 个任务 5 个模型全部 100%**。

`*` Qwen 的分数被本测量低估，见下。

## ⚠️ 尺子缺陷（必须标注，否则这份数据会误导）

Qwen 在 `extract-amounts` 上 0%，而其余 6 任务全 100% —— 一枝独秀的失败，先查尺子：

- 三轮**全部 compl=512**（撞我设的 max_tokens 上限），content 空，回退取 reasoning 尾部——
  而尾部混进了推理过程的杂数字（`4,5,56`）。**我的 `max_tokens=512` + 推理尾回退把它判低了。**
- 但也非纯尺子问题：max 提到 1024 仍错（幻觉"3"、漏"12"）——思考开时它过度推理、带入杂数。
- **关闭思考（`enable_thinking=False`）：18 token，一次答对。**

结论：**Qwen 的 86% 是本测量的下限，不是它的能力上限。** 要给它一个干净的数，需重跑于思考关或更大 max_tokens。

## 站得住的三条结论

1. **纯文本任务上，主流国产模型近乎满分。** GLM-5.2 / DeepSeek-V4-Pro / Kimi-K2.6 三个 100%，
   LongCat 95%。唯一有区分度的任务是 `extract-amounts`（抽取+去重，比分类难）。分类/格式/JSON 抽取都是满分。

2. **真正的成本变量是"推理开销"，不是模型档次。** 同为答对，completion 中位从 DeepSeek **40** 到 Qwen **426**——**10× 差**。
   纯文本任务不需要推理，推理模型开着思考纯烧 output token（output 是最贵的部分）。

3. **可操作的省钱杠杆：纯文本任务关掉思考。** Qwen 实测 512 → 18 token（**28×**），且从错变对。
   这条比"选哪个模型"影响大得多。

## 与之前 coding 测试的对照（方法学）

- **这次消除了最大的方法学缺陷**：同端点 = 无 harness = 无人格污染（还记得 codex 被 superpowers 带跑吗）。
  跨模型可比性从"苹果对橘子"变成真·同条件。
- 纯文本任务方差也比 coding 小（分类/格式类几乎全对），n=3 的结论比 coding 那批更可信。
- **但仍不是专业 benchmark**：7 任务、n=3、单一供应商中转（硅基流动，非各厂官方端点，可能有中转层差异）。

## 本数据【不能】支持的

- 不能给模型排名（样本太小；Qwen 的分被尺子低估）。
- 不能推广到 coding / agentic / 长上下文任务——这里只测了纯文本单轮。
- 硅基流动是第三方聚合端点，非各厂官方 API，token 计数/行为可能与官方端点有差异。
- `enable_thinking` 是否所有模型都支持未逐一验证（Qwen 确认可用）。

## 由本测试新增/验证的 skill 规则

- 路由第 2 步已有「纯文本走裸 API/aichat」；**补充：纯文本任务在支持的模型上关闭思考**（`enable_thinking=false` 或等价），省 10–100× output token。
- verifier 对推理模型要给足 max_tokens，或按 content-only 判定（reasoning 尾回退不可靠）——写入测量纪律。
