# 实测证据汇编（SKILL.md 各规则的数据出处）

本文件汇编 SKILL.md 各条规则背后的实测数据与论证细节。**读前先知道边界**：样本量普遍很小（n=3–5、任务多为自造），所有百分比应读作**方向性证据**，不是精确测量；任务自造只缓解、不能证明消除训练集污染；深浅两组任务是不同的题，任务身份本身是混杂变量。原始数据、任务定义与判分脚本在仓库 `bench/` 下，可复现复核。

## thinking 深浅实测（thinking 是能力旋钮的依据）

设置：5 模型 × 10 任务 × n=5，裸 API，两组任务都有唯一可验证答案，唯一操纵变量是搜索深度。完整数据见 `bench/RESULTS-DEPTH-20260710.md` + `bench/RESULTS-RIGOR-20260710.md`。

| 深度 | 关思考 | 开思考 | 净效应 |
|---|---|---|---|
| 浅 | 99% [96,100] | 100% [98,100] | **+1pp（CI 与 0 重叠 = 无增益），40× token** |
| 深 | **28%** [20,37] | **99%** [95,100] | **+71pp（CI 完全不重叠），120× token** |

- **5/5 模型在深任务上均为正增益**（同向性比均值更稳健）。
- 深度边界比直觉靠前：递推题 k=1（单次乘加取模）关思考仅 56%，k≥2 全线 0%；纯文本多跳定位（答案字面在输入里）单跳 100%、两跳 67%、四跳 22%。
- **塌方无警告信号**：72 次失败全部是 `wrong`，零空输出、零拒答。11 步递推关思考 25/25 全错（输出 988/916/470，正确 256），用 1–9 个 token、格式完美、自信满满。
- 3 连发探针（关思考同任务跑 3 次，不一致 ⇒ 深任务）：灵敏度 85%、特异度 93%，漏网格全部是"一致且答对"的无害情形，零个"稳定的假数字"漏网。样本与判分见 `bench/RESULTS-RIGOR-20260710.md`。
- **官方 CLI 与裸 API 行为不同**（RESULTS-RIGOR ⑦）：Claude/Codex 的 effort 是温和成本旋钮——haiku"关思考"会把推理写进正文自救（83%），codex low 仍在推理（75%）；28% 塌方是**裸 API + `enable_thinking=false` + 链式任务**特有。
- 曾声称"开思考会把对的想成错的"，已撤回（预算给够时两边都对，见 `FINDINGS-thinking-truncation.md`）；`completion == max_tokens` 是截断指纹，该次调用无效而非答错。

## 审查任务的缺陷深度实测

两轮植入缺陷实测（RESULTS-RIGOR ⑤⑥）：

- **单节内硬伤**（数值自相矛盾、模板化泄漏）= 浅操作：关思考召回 89% vs 开 85%（23× token 零回报）；思考还可能收紧阈值弄丢判断型缺陷（LongCat 开思考 6/6 漏报全是"多重比较/因果跳跃"）。三家便宜模型关思考取并集 = 100%。
- **跨节一致性缺陷**（划分数 vs 样本数、df vs 分组、跨节单位基准）= 深操作：思考净效应 +21pp（67%→88%）；off 并集只有 89% 封不了顶、on 并集 100%——深缺陷上"多想"与"多家"互补不互替。
- 跨节**计数**核对（"6 种" vs "8 种"）连开思考也只有 3/8——交给人工或让模型写脚本数。

## 档位实测（档位是成本旋钮的依据）

- **Anthropic 内部三档**（4 硬任务 × 3 轮，`bench/RESULTS-TIERS-20260708.md`）：haiku 仅在 51 步精确递推（模型必须亲自算）上 67%，sonnet/opus 100%；其余三个硬任务三档全 100%。该唯一例外落在已被铁律禁止的派发方式上。
- **codex 三档**：全部四个硬任务 low 与 high 持平。
- **美元核实**（Anthropic 官方单价，4 任务合计）：haiku $0.144 / sonnet $0.426 / opus $0.404——最低档便宜约 3 倍。haiku 输出 token 是 opus 的 12.6 倍，但单价低 5 倍且 fresh 几乎全被缓存吸收，话痨没吃掉价差。
- **中档 thrash**：sonnet 在需要多轮工具调用的任务上 thrash 到 9 轮 / 47.6 万 input，opus 仅 2 轮 / 10.4 万。美元排序随定价翻转（Sonnet 5 引导价 $2/$10 至 2026-08-31，之后需复核），**token 行为的非单调是稳健的**。

## 「自算不是厂商属性」实测

同一道 51 步递推题：硬基准里 GLM 3/3、StepFun 1/3；独立探针里 GLM 1/3、StepFun 2/3——完全反转，合计都接近抛硬币。对照 `tools` 字段：答对的那几次模型写了代码去跑，答错的那几次它心算了。探针 prompt 里"你可以心算、手算或写代码计算，方式不限"正是邀请失败的元凶。StepFun 单独证据：写校验和代码 100%，亲自迭代 33%，肉眼数 assert 33%（数成 17/18），step-mid 同为 33%——升档救不了。

## 跨厂商独立性量化

- 500 次调用实测：同一模型对同一题错两次、错成同一个值的概率 6%；两个不同厂商 1%——交叉验证被"共识错误"骗过的概率约为自查的 1/6。
- 错误**位置**高度相关（逐任务错误率 Pearson 中位 0.79，都栽在难任务上）；独立的是错误的**值**，交叉验证吃的正是这一点。
- **蒸馏/同源收敛边界**：跨厂商独立性对采样型错误（算错、抄错）稳健，对学来的错误（共享误念、对歧义的相同偏好解读）失效。实测：五家模型对同一道歧义题给出逐字相同的解读——语义解读层已高度趋同，投票在该层近乎无效。
- **独立性可监测**：共识错误率逐模型对测量，10 对中 9 对为 0，唯 GLM×LongCat 3/90（应视为部分同源）。
- **反常一致的首例**：5 模型在"错误答案"上完全一致，排查后是基准自己的病态题（题面未声明取整，模型按小数忠实作答被误判）——一致到反常时先怀疑自己的题。

## harness 税与缓存

同一句"你好"的 input：裸 API 无系统提示地板 11 token → 加系统提示 241 → `claude -p` ~30k（工具定义 + 技能索引）。同一模型同一空任务，内部 subagent 对比外部 `claude -p` 相差 7.6×。完整数据表见 `references/dispatch-design.md`。计费双峰：冷调用付全额 fresh（16k–30k），热调用几乎全走缓存（0.1k–3k，TTL 约 5 分钟）。技能索引有上限（codex 约 2% 上下文预算）：移除 14 个技能后模型可见 prompt 几乎不变（42,802→43,052 字符）——"少装 skill 省钱"是错的。

## advisor 经济学在 CLI 架构下反转

原始依据（Anthropic advisor 工具基准）：Sonnet+Opus 顾问比 Sonnet 单独高 2.7pp 且成本降 11.9%。但原生 advisor 是 Messages API 的 in-request 工具、不重发上下文；跨 CLI 咨询每次付完整 20k–30k 冷启动。本项目实测 advisor 臂 input 2.1×、耗时 2.2×、正确率零增益（执行者本就 100% 且零 thrash，天花板效应——advisor 的准确率收益既未复现也未证伪）。

## 框架污染实证（盲验分级的依据）

512 截断假象在项目内活了一整天，所有复查都在"overthinking"框架里打转；抓出它的是零上下文的外部模型——没看到数据、没继承术语，攻击的是前提而不是细节。这是"编排者写 prompt 会传染项目框架"的直接证据，也是隔离分级第 3/4 级存在的理由。
