# 密钥安全铁律（可审计）

本 skill 会接触 API key（读 cc-switch、写 aichat 配置、派发注入）。以下六条不可违反，也是公开分享让人敢用的前提——**任何人读本文件即可确认它对密钥只做"本地只读 + 按进程注入"，不外传、不留痕**。

1. **只读本地、绝不外传**：读配置/密钥只在本机；key 只作为认证信息由 CLI 传给用户指定的那个模型端点，skill 自身绝不把 key 发往任何其他地址。
2. **绝不回显**：输出/日志/报告/留痕文件里 key 一律打码（如 `<len49>`），不打印明文。
3. **绝不落盘到本 skill 写的文件**：manifest、`.dispatch/` 留痕等只记"来源 / 是否可提取"，绝不写入 key 本身。
4. **绝不进模型上下文**：key 只注入子进程环境变量，绝不拼进发给任何模型的任务文本（否则泄露给被派发的模型，且随上下文发往宿主 agent 的云端）。
5. **引用优先于复制**：优先用用户级环境变量引用（`$XXX_KEY`）或派发时临时提取，不把密钥复制到新位置。
6. **读凭据库需用户知情**：读 cc-switch 等凭据库前告知用户，不静默。

## 为什么这些规则比"本地运行"更重要

"全在本地"**不等于**杜绝泄露：

- key 必然要发到它自己的 provider（预期收件人，不可避免）。
- **头号风险**：执行本 skill 的 agent 本身是**云端模型**——一旦 key 被回显进命令、或含 key 的配置被读入上下文，它就随上下文发到了模型厂商服务器 + 落进本地 transcript。**"本地运行"救不了这个**，只有第 2、4 条纪律能防。
- 本地配置文件（cc-switch db、settings.json、env 变量、aichat config）都是明文静态存储，任何本机进程可读。

## 代码级落实

`cc_switch.py` 是铁律的代码化：

- `list` 模式输出 provider 清单，**token 字段从不输出**，只给 `has_token` 布尔。
- `exec` 模式在**脚本子进程内部**读取 token 并注入 `subprocess` 的 env，token 从不打印、从不回到调用它的 agent 的上下文。
- 全程 `sqlite3` 只读模式（`file:...?mode=ro`），绝不修改 cc-switch 数据，绝不使用其"全局切换"机制（那会写 `~/.claude/settings.json`，污染宿主官方登录）。

## 覆写隔离（实测依据）

`ANTHROPIC_BASE_URL` / `ANTHROPIC_AUTH_TOKEN` **只能作为子进程环境变量临时传入**，绝不写进 `~/.claude/settings.json` 的 env 或任何全局配置——一旦写全局，所有 claude 调用（含官方订阅的宿主会话）都会被重定向到第三方端点，这是最常见的冲突根因（第三方"一键助手"常这么干）。

实测（claude 2.1.204，2026-07-08）：子进程覆写时 CLI 明确以注入 auth 源**优先于 claude.ai 登录**；宿主会话登录、`settings.json`（env 保持 `{}`）、宿主进程环境三者均不受影响；被重定向的子进程与走官方订阅的子进程可**并发共存、互不干扰**。隔离是操作系统进程级的。

## 工作区隔离（被派发模型只读）

密钥之外的第二个面：被派发的子进程**继承宿主 cwd**，几个模型站在同一个工作区里。本 skill 的设计是**单写者**——外部模型只出意见，落盘一律由宿主 agent 执行。因此派发时必须显式剥夺写能力，而不是依赖"它大概不会写"。

- `codex exec` 一律带 `-s read-only`。
- `claude -p` 一律带 `--tools Read,Grep,Glob`（纯文本任务用 `--tools ""`）。`cc_switch.py` 已代码化这条默认。
- **`--tools` 在工具注册层生效，`--permission-mode bypassPermissions` 也压不过它**（claude 2.1.205 实测）；`--permission-mode` 本身没有只读档，不可用作护栏。
- 不加参数时默认 `-p` 虽也拒写，但靠的是"非交互无法确认"，且被拦截时 **exit 0**、把拒绝当正常答案返回。**不可依赖。**

这条纪律同时解释了本 skill 为何不需要 git worktree：worktree 解决的是多个**可写** agent 并发，而交叉验证要求多个模型看**同一份**输入才谈得上比对分歧——给它们各自一个 worktree 反而破坏语义。

## 分享注意

以上是**规则**，约束 agent 执行时的行为，不是代码级强制（`cc_switch.py` 部分做到了代码级）。规则能保证"正确使用本 skill 时安全"，但挡不住有人拿去改成恶意版——那已是另一个 skill。**建议只从可信来源获取本 skill，避免运行被篡改的副本。**
