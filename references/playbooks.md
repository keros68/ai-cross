# 编排方案复用（可选）

**默认不用。** 主 agent 每次自主决策派发方案即可——大多数用户不需要复用机制。本文件只为重复性强的工作流留一个零成本的口子。

## 保存

用户说「保存本次编排」时，把本次的：

- 任务模式（什么类型的任务触发这套编排）
- 派发矩阵（哪一路用什么档位、什么角色、哪个厂商）
- 轮次结构（几轮 review、什么条件下升级/熔断）
- 验证方式（怎么判定"过了"）

写成 `<本skill目录>/playbooks/<名字>.md`。

## 复用

用户说「按方案 `<名字>` 执行」→ 读该文件照做。

**关键：模型 ID 仍取 manifest 的实时值，不用方案里记录的旧 ID。** 方案记录的是「档位 + 厂商 + 角色」这类稳定语义，不是具体型号——型号会漂（见 `channels.md` 末节）。

## 示例结构

```markdown
# playbook: coding-双厂商闭环

## 适用
需要交叉验证的功能实现类任务

## 派发矩阵
| 路 | 角色 | 档位 | 厂商约束 |
|---|---|---|---|
| 1 | 实现 | 中 | 任意 |
| 2 | 审查 | 高 | 必须 ≠ 实现方厂商 |

## 轮次
实现 → 本地跑验证 → 审查 → 修正 → 重跑验证 → 复审，≤3 轮
任一通道连续失败 2 次 → 熔断切同档备选

## 通过判据
审查方给出 PASS 且本地全部测试通过
```
